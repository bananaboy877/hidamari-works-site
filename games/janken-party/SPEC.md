# どうぶつ運動会 - 仕様書

## コンセプト
じゃんけんコントローラー（3ボタン）だけで遊べる、動物園の運動会をテーマにしたリハビリ向けミニゲーム集。
高齢者でも迷わず楽しめるシンプルな設計。

## ターゲット
- リハビリ患者（高齢者含む）
- 手指のリハビリ（握力・指先運動・眼球追従・注意力・判断力・記憶力）

## 入力デバイス

### じゃんけんコントローラー（XIAO RP2040ベース）
| ボタン | キー | 色   | 表示 |
|--------|------|------|------|
| グー   | A    | 赤   | ✊   |
| チョキ | S    | 黄   | ✌️   |
| パー   | D    | 青   | ✋   |

### マウス
- 「もどる」ボタンなどのUI操作のみ
- じゃんけんコントローラーに「戻る」機能は持たせない

## 画面仕様
- **解像度**: 1920×1080px 固定、スクロールなし
- **フォントサイズ**: 最小28px、通常36px、見出し56px以上
- **テーマカラー**: 動物園・運動会らしい明るくポップな配色
- **HTMLファイル + 音声フォルダ**: フォルダ単位でオフライン配布

## 画面遷移

```
タイトル画面（どうぶつ運動会）
  │  任意のボタンでスタート
  ▼
ゲーム選択画面
  │  まだ遊んでいないゲームからランダムに1つ選ばれる
  │  （全9種すべて遊んだらリセット）
  │  ゲーム紹介アニメーション表示後、自動的にゲーム開始
  ▼
ミニゲームプレイ
  │
  ▼
リザルト画面
  │  スコア表示 + 動物メダル授与演出
  │  通算メダル数を表示
  │  「つぎのゲームへ」→ ゲーム選択に戻る
  │  「おわる」(マウスクリック) → タイトルに戻る
  ▼
（ループ）
```

### ゲーム選択の仕組み
- プレイヤーがゲームを選ぶ必要はない（選択負荷ゼロ）
- まだ遊んでいないゲームの中からランダムに1つ選出
- 9種すべて遊んだら「全部遊んだ！」演出後リセット
- 選出されたゲームの紹介画面を表示:
  - 動物キャラ＋ゲーム名＋ルール説明テキスト
  - **同時にTTS音声でルール説明を読み上げ**（あらかじめ用意した音声ファイル）
  - 音声再生完了後、「任意のボタンを押してスタート」でゲーム開始

## 動物メダルシステム

### 概要
- 各ミニゲームのスコアに応じて動物メダルを獲得
- 通算メダル数を画面上部に常時表示
- 現時点ではメダルに集める以外の機能は持たせない（将来拡張用）
- localStorageに保存（ブラウザを閉じても記録が残る）

### メダル獲得基準（各ゲーム共通ランク）
| ランク | メダル数 | 条件 |
|--------|---------|------|
| 金     | 3枚     | スコア上位（各ゲームで個別に設定） |
| 銀     | 2枚     | スコア中位 |
| 銅     | 1枚     | ゲームクリア |

### メダルの見た目
- 動物の顔が刻まれたメダル（ライオン、ゾウ、キリンなど）
- 金・銀・銅で色が変わる
- 獲得時にキラキラ演出

## ミニゲーム一覧（全9種）

### 1. リズムでポン！
- **動物**: パンダ
- **ジャンル**: リズム
- **リハビリ効果**: タイミング感覚、指の協調運動
- **ルール**: 画面上部から降ってくるノーツ（グー/チョキ/パーのアイコン）が判定ラインに重なった瞬間に対応ボタンを押す
- **スコア**: Perfect/Good/Miss で加点。コンボボーナスあり
- **時間**: 約60秒（1曲分）
- **メダル基準**: 金=90%以上Perfect、銀=70%以上Good以上、銅=クリア

### 2. おぼえてジャンケン
- **動物**: ゾウ
- **ジャンル**: 記憶
- **リハビリ効果**: 短期記憶、順序記憶
- **ルール**: 画面に表示される手の順番（例: グー→パー→チョキ）を覚えて、同じ順番で入力。正解するたびに1つずつ長くなる
- **スコア**: 正解した最長の長さ
- **メダル基準**: 金=8つ以上、銀=5つ以上、銅=3つ以上

### 3. はやおしバトル
- **動物**: チーター
- **ジャンル**: 反射
- **リハビリ効果**: 反応速度、視覚認識
- **ルール**: 画面中央に大きく表示される手を見て、対応するボタンをできるだけ早く押す。10問連続
- **スコア**: 平均反応時間
- **メダル基準**: 金=平均0.5秒以下、銀=平均0.8秒以下、銅=完走

### 4. もぐらたたき
- **動物**: モグラ（そのまま）
- **ジャンル**: 反射・注意
- **リハビリ効果**: 注意力、視野の広さ、反射神経
- **ルール**: 3つの穴（左=グー、中=チョキ、右=パー）からモグラが顔を出す。対応ボタンで叩く。間違いボタンはミス
- **スコア**: 叩けた数 - ミス数
- **時間**: 45秒
- **メダル基準**: 金=25匹以上、銀=15匹以上、銅=8匹以上

### 5. 連打チャンピオン
- **動物**: ゴリラ
- **ジャンル**: 連打
- **リハビリ効果**: 握力・指の運動量・持久力
- **ルール**: 画面に指定されたボタン（ラウンドごとに変わる）をひたすら連打。3ラウンド制
- **スコア**: 合計連打回数
- **時間**: 各ラウンド10秒（計30秒）
- **メダル基準**: 金=60回以上、銀=40回以上、銅=20回以上

### 6. スロットでドン！
- **動物**: サル
- **ジャンル**: タイミング
- **リハビリ効果**: タイミング感覚、集中力
- **ルール**: 3つのリールが回転。A→S→Dの順にボタンを押して停止。絵柄が揃えば高得点。5回挑戦
- **スコア**: 揃った回数×絵柄ポイント
- **メダル基準**: 金=3回以上3つ揃い、銀=1回以上3つ揃い、銅=2つ揃い達成

### 7. じゃんけんウォーズ
- **動物**: ライオン
- **ジャンル**: 戦略・じゃんけん
- **リハビリ効果**: 判断力、思考力
- **ルール**: CPUとじゃんけん対決。勝つとダメージ、負けると相手にダメージ。相手のHPを0にすれば勝ち。ヒント演出あり（相手の癖が出る）
- **スコア**: 残りHP × 勝利ターン数ボーナス
- **メダル基準**: 金=HP80%以上で勝利、銀=HP50%以上で勝利、銅=勝利

### 8. どうぶつクイズ
- **動物**: フクロウ
- **ジャンル**: 知識
- **リハビリ効果**: 思考力、読解力
- **ルール**: 動物にまつわる3択クイズ。A/S/Dで回答。10問出題
- **スコア**: 正解数
- **メダル基準**: 金=9問以上正解、銀=6問以上正解、銅=3問以上正解

### 9. かけっこレース
- **動物**: ウサギ
- **ジャンル**: 交互連打
- **リハビリ効果**: 指の交互運動、協調性
- **ルール**: A→S→D→A→S→D…の順番通りにボタンを押してキャラを走らせる。順番を間違えるとつまずく。ゴールまでのタイムを競う
- **スコア**: ゴールタイム
- **メダル基準**: 金=15秒以内、銀=25秒以内、銅=ゴール

## UI/UX 設計方針

### フォント
- ポップで読みやすい日本語フォント（例: Zen Maru Gothic）
- 漢字を普通に使用（高齢者は漢字が読める。ふりがな不要）
- 高齢者が読みやすい太字ウェイト

### 配色
- 背景: 明るい空色〜芝生グリーンのグラデーション（運動会の雰囲気）
- ボタン: 赤(グー) / 黄(チョキ) / 青(パー) で統一
- テキスト: 濃い茶色 or 黒（コントラスト確保）
- アクセント: 金色（メダル）、白（雲・装飾）

### アニメーション
- ゆったりめの演出速度（高齢者が追える速度）
- ボタン押下時のフィードバック（拡大＋色変化）
- メダル獲得時のキラキラ演出
- 画面遷移はフェードイン/アウト

### アクセシビリティ
- コントラスト比 4.5:1 以上
- 色だけに頼らない（形状・テキストでも識別可能）
- 点滅は控えめ（光過敏症対策）

## 技術仕様

- **形式**: 単一HTMLファイル（Vanilla JS）
- **描画**: CSS + Canvas（ゲーム部分）or DOM操作
- **フォント**: Google Fonts（Zen Maru Gothic）
- **保存**: localStorage（通算メダル数、各ゲームのハイスコア）
- **画像**: CSS/SVGで描画（将来的に画像に差し替え可能。必要時は画像生成プロンプトをHTMLコメントに記載して引き渡し）
- **音声**: 外部ファイル管理。`audio/` フォルダにMP3を配置し、HTMLから相対パスで参照
- **配布形態**: `janken-party/` フォルダごと配布（index.html + audio/）
